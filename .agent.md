# Keramikas Darbnīca Development Agent

You are assisting with **Keramikas darbnīca**, a Latvian language e-commerce platform selling handmade ceramics. This is a student-friendly project using vanilla JavaScript, Supabase, Cloudflare Pages, and Klix payments.

## Project Context

### Architecture
- **Frontend**: Pure HTML/CSS/JavaScript (no build tools, no npm)
- **Database**: Supabase (PostgreSQL with RLS policies)
- **Hosting**: Cloudflare Pages + Cloudflare Functions
- **Payments**: Klix payment gateway integration
- **Language**: Latvian UI and documentation; English code comments

### Key Features
1. **Multi-dimensional product organization**:
   - Clay origin (Latvian map visualization)
   - Glazing/firing styles
   - Artistic themes
   - Limited quantities per work (tracks remaining inventory)

2. **Three main user roles**:
   - Public customers (shop, cart, checkout)
   - Administrators (CMS, product management, orders)
   - System (Cloudflare Functions for payments/emails)

3. **Key integrations**:
   - Supabase Auth for admin login
   - Klix for secure payment processing
   - Email notifications after successful orders
   - Package delivery tracking

## Development Guidelines

### File Organization
```
/          → HTML pages (shop, product, cart, checkout, admin, legal, index)
/css/      → Style sheets (vanilla CSS, dark artistic theme)
/js/       → Client-side logic (no external deps except Supabase)
/functions/api/ → Cloudflare Functions (server-side: Klix, emails, webhooks)
/sql/      → Database schemas, RLS policies, seed data
/docs/     → Setup guides and documentation (in Latvian and English)
```

### Important Files by Purpose

**Configuration & Setup**
- `js/config.example.js` → Student must copy to `js/config.js` with Supabase credentials
- `docs/SUPABASE_SETUP.md` → Database initialization
- `docs/KLIX_SETUP.md` → Payment gateway setup
- `docs/CLOUDFLARE_SETUP.md` → Deployment instructions

**Core Frontend Systems**
- `js/supabaseClient.js` → Supabase initialization (anon key only, never service role)
- `js/products.js` → Data queries for products
- `js/cart.js` → localStorage-based shopping cart
- `js/checkout.js` → Order form + package selection UI
- `js/admin.js` → Admin CMS (create/edit products, manage inventory, view orders)

**Backend Functions**
- `functions/api/create-klix-purchase.js` → Creates order + initiates Klix payment
- `functions/api/klix-webhook.js` → Validates payment completion, reduces inventory
- `functions/api/send-order-email.js` → Sends order confirmation emails

**Database**
- `sql/schema.sql` → Core tables: products, orders, admin_users, storage
- `sql/functions.sql` → Triggers and stored procedures
- `sql/rls-policies.sql` → Row-Level Security for data protection
- `sql/seed-demo-data.sql` → Demo data for testing

### Code Standards

**JavaScript**
- Vanilla JS only (no jQuery, React, Vue, etc.)
- ES6 modules when needed (imported from CDN)
- Comments in English; UI strings in Latvian
- Async/await preferred over callbacks
- Simple, readable code suitable for student learning

**Database**
- All queries run through Supabase client (respects RLS)
- Never expose service role key in frontend code
- Admin operations require authentication via Supabase Auth
- Inventory automatically reduced via webhook trigger

**Security**
- Service role key only in Cloudflare Functions (server-side)
- Public/anon key only in frontend
- RLS policies enforce authorization per user role
- Sensitive data (admin passwords, payment tokens) never logged

## Common Tasks

### Adding a New Product
1. Upload image to `product-images` bucket via admin panel
2. Fill product form: name, description, price, quantity, clay origin, glazing style, theme
3. Select thumbnail image
4. Data saved to `products` table with RLS enforcement

### Customizing Styling
- Edit `/css/style.css` for main theme (dark, artistic aesthetic)
- Edit `/css/admin.css` for admin interface
- Maintain consistency across pages

### Testing Workflows
1. **Product workflow**: Add → view in shop → add to cart → checkout
2. **Admin workflow**: Login → create/edit product → view orders
3. **Payment workflow**: Uses Klix test environment; order stored before payment confirmed
4. **Email delivery**: Check logs in Cloudflare dashboard

### Deployment
1. Push to GitHub
2. Cloudflare automatically deploys from GitHub Actions
3. Add environment variables in Cloudflare dashboard (Supabase URL, Klix credentials)
4. Functions deploy automatically with Pages

## When Helping with This Project

✅ **DO**:
- Explain code in simple terms suitable for students
- Point to relevant SQL/JS files by path
- Consider performance (students may not optimize prematurely)
- Suggest improvements that maintain simplicity
- Provide Latvian translations for UI text when needed
- Test across the full workflow (product → cart → payment → email)

❌ **DON'T**:
- Suggest adding npm, build tools, or complex frameworks
- Expose service role keys anywhere in code
- Store sensitive data in localStorage or cookies
- Bypass RLS policies (enforce through Supabase, not frontend)
- Assume security practices are in place (audit them)
